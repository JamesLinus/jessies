Edit

Introduction

Edit is an integrating (rather than integrated) development environment for programmers, written in Java. Its most distinctive characteristic is its use of tiled (i.e. non-overlapping) subwindows for editing multiple files.

Using Edit

Starting Edit

The edit shell script or edit.bat batch file can be used to start Edit; all it does is invoke java with the main class e.edit.Edit. Edit will open a window whose size can be configured in the file edit.properties.

Keyboard Shortcuts

The following hot-key combinations are availble (where C means Ctrl for PC users, and Ctrl or Command for Mac users):

C-FFind...
C-DFind previous match
C-GFind next match
 
C-LGoto Line...
 
C-SSave
 
C-XCut
C-CCopy
C-VPaste
 
C-BBuild project
 
C-ZUndo
S-C-ZRedo
 
C-WClose window

Finding Text

The 'Find' window (C-F) opens with the current selection. You can type a new search string for interactive searching. Pressing C-D or C-G in a text window will find the previous or next matches of the currently selected text, without opening a find window.

Going to a Line

The 'Goto' window (C-L) opens with the current line number. You can type a line number and press Return to go to that line.

Moving and Closing Windows

Windows can be moved by dragging their titlebars.

Windows can be closed by clicking the little crosses in their titlebars. A red cross means that the file has been edited since it was last saved. Clicking a red cross will turn it black without saving, and indicates that you don't care about your changes. Clicking a black cross will close a window.

Double-clicking on a titlebar will expand that window at the expense of the others in the column.

Building your Project

The C-B key combination clears the current workspace's errors window, and begins searching upwards from the current context until a Makefile or build.xml is found, at which point make or ant is invoked in that context.

Using Multiple Workspaces

Along the left-hand edge of the Edit window there are four tabs, labelled 'alpha', 'beta', 'gamma' and 'delta'. Clicking on one of these switches to a new workspace: a fresh Edit where you can work on a different project or whatever. This is sometimes more convenient than running multiple copies of Edit, or opening files from multiple projects in the same Edit workspace.

Configuring Edit

Edit has various configuration options, all of which can be set in edit.properties. This file is read at start-up. It's a plain text file which can be edited with Edit itself, and which should be self-explanatory.

If you don't want to use Internet Explorer to open URIs, you will need to tell Edit the location of your preferred browser.

You will need to tell Edit the location of your JDK documentation to use the JavaDoc feature.

Extending Edit

The tools on the tools menu are read from the properties file. Edit sets a few environment variables to communicate with your code. The variable EDIT_CURRENT_FILENAME is the currently selected text window's filename. The variable EDIT_WORKSPACE_ROOT is set to the root directory of the current workspace.

Downloading Edit

Edit is available as edit.tgz, a file containing both the source and the .class files. You'll need to untar everything before you can run Edit.
